> 生成日期: 2026-07-17
> 用途: 记录项目结构、API 接口、数据模型、页面路由关系，方便后续开发

## 一、项目总览

**技术栈**:
- **前端**: Vue 2.6.14 (部分页面) + 原生 JS (admin-waterfall.html)
- **图标库**: Font Awesome 6.5.0
- **运行时**: Cloudflare Pages (主要) / Node.js + Docker (server/)
- **存储**: Cloudflare KV + R2 + Telegram + Discord + S3 + WebDAV + GitHub + HuggingFace
- **构建**: 无构建步骤，纯静态 HTML

**部署模式**:
1. **Cloudflare Pages** — `functions/` 目录下 Cloudflare Workers 风格的 Serverless Functions
2. **Docker / Node.js** — `server/` 目录下 Express.js + SQLite 服务端

## 三、前端页面内部结构

### 3.1 `index.html` — 文件上传主页

**Vue 数据 (data)**:

**主要方法**:

**Vue 数据 (data)**:
- `activeTab` — 当前标签: `files | users | groups | sections`
- `tableData[]` — 文件列表
- `selectedFiles[]` — 选中的文件
- `adminList[]` — 当前标签下的数据列表
- `adminDataCache` — 缓存的管理数据 `{ users, groups, userGroups, sections, sectionPermissions }`
- `adminForm` — 模态框表单对象
- `builtinPermOptions[]` — 内置权限选项 (9项)

**关键方法**:

- `getGroupBuiltinPerms(groupId)` — 获取身份组内置权限列表

- `checkConnectionStatus()` — 检查存储连接状态

### 3.3 `gallery.html` — 图库浏览

## 四、后端 API — Cloudflare Functions

所有 API 位于 [`functions/`](../functions/) 目录下。

### 4.1 中间件链

```
请求 → functions/_middleware.js → api/_middleware.js → api/admin/_middleware.js
                                         ↓                       ↓
                                    errorHandling           ensureDefaultAdmin
                                    telemetryData           checkAuthentication
```

| 中间件 | 文件 | 功能 |
|--------|------|------|
| `errorHandling` | [`functions/utils/middleware.js`](../functions/utils/middleware.js) | 全局错误捕获 |
| `telemetryData` | [`functions/utils/middleware.js`](../functions/utils/middleware.js) | 请求遥测 |
| `checkAuthentication` | [`functions/utils/auth.js`](../functions/utils/auth.js) | Cookie/Basic Auth 验证 |
| `ensureDefaultAdmin` | [`functions/utils/admin-data.js`](../functions/utils/admin-data.js) | 首次启动时创建默认管理员 |

### 4.2 认证 API

| 端点 | 文件 | 方法 | 说明 |
|------|------|------|------|
| `/api/auth/login` | [`login.js`](../functions/api/auth/login.js) | POST | 用户登录，返回 Set-Cookie |
| `/api/auth/login` | [`login.js`](../functions/api/auth/login.js) | GET | 检查是否需要认证 |
| `/api/auth/check` | [`check.js`](../functions/api/auth/check.js) | GET | 检查当前会话有效性 |
| `/api/auth/logout` | [`logout.js`](../functions/api/auth/logout.js) | POST | 登出，删除会话 |

**认证流程**:
1. 用户提交用户名+密码 → `POST /api/auth/login`
2. 服务端验证凭据 → 创建 HMAC 签名会话 → 存入 KV `session:{token}`
3. 返回 `Set-Cookie: k_vault_session={token}`
4. 后续请求验证 Cookie → `checkAuthentication()` 从 KV 读取会话

## 五、数据模型

### 5.1 KV 存储 — `admin:data`

存储在 Cloudflare KV (binding: `img_url`)，key 为 `admin:data`。

```javascript
{
  users: [
    {
      id: "user_1748394028319",      // 用户唯一 ID
      username: "admin",              // 登录名
      nickname: "管理员",             // 显示名称
      passwordHash: "sha256hex...",  // SHA-256 密码哈希
      role: "admin" | "user",        // 角色
      enabled: true,                 // 是否启用
      createdAt: 1748394028319       // 创建时间戳
    }
  ],
  groups: [
    {
      id: "group_1748394028319",     // 身份组 ID
      name: "管理员组",               // 组名
      description: "",               // 描述
      builtinPerms: [                // 内置权限数组 (2026-07 重构)
        "upload", "read", "download", "manage", "delete",
        "create_user", "create_admin", "create_group", "modify_group"
      ]
    }
  ],
  userGroups: [
    { userId: "user_xxx", groupId: "group_xxx" }   // 用户-身份组关联
  ],
  sections: [
    {
      id: "section_1748394028319",   // 版块 ID
      name: "技术讨论",               // 版块名称
      slug: "tech",                  // 唯一标识
      description: ""                // 描述
    }
  ],
  sectionPermissions: [
    { sectionId: "section_xxx", groupId: "group_xxx", level: "read" | "write" }
    // 版块权限 (仅保留兼容，不再用于新功能)
  ]
}
```

### 5.2 文件 KV 存储

文件使用 `img:{id}`, `vid:{id}`, `aud:{id}`, `doc:{id}` 前缀存储，metadata 包含:

```javascript
{

  TimeStamp: 1748394028319,
  fileSize: 1024000,
  storageType: "telegram" | "r2" | "s3" | "discord" | "huggingface" | "webdav" | "github",

  liked: false,                        // 是否收藏
  ListType: "White" | "Block",         // 白名单/屏蔽
  Label: "adult",                      // 标签
  // 存储后端特有字段:
  telegramFileId: "...",               // Telegram
  r2Key: "...",                        // R2
  s3Key: "...",                        // S3
  discordChannelId: "...",             // Discord
  discordMessageId: "...",
  hfRepoId: "...",                     // HuggingFace
  hfFilePath: "...",
  githubOwner: "...",                  // GitHub
  githubRepo: "...",
  githubFilePath: "...",
  webdavUrl: "...",                    // WebDAV
}
```

### 5.3 SQLite 数据模型 (Docker/Node.js 服务端)

**文件**: [`server/db/schema.sql`](../server/db/schema.sql)

```sql
-- 用户表
CREATE TABLE users (
  id TEXT PRIMARY KEY,
  username TEXT UNIQUE NOT NULL,
  nickname TEXT DEFAULT '',
  password TEXT NOT NULL,
  role TEXT DEFAULT 'user',
  enabled INTEGER DEFAULT 1,
  created_at INTEGER NOT NULL,
  updated_at INTEGER NOT NULL
);

-- 身份组表
CREATE TABLE groups (
  id TEXT PRIMARY KEY,
  name TEXT NOT NULL,
  description TEXT DEFAULT '',
  created_at INTEGER NOT NULL,
  updated_at INTEGER NOT NULL
);

-- 版块表
CREATE TABLE sections (
  id TEXT PRIMARY KEY,
  name TEXT NOT NULL,
  slug TEXT UNIQUE NOT NULL,
  description TEXT DEFAULT '',
  created_at INTEGER NOT NULL,
  updated_at INTEGER NOT NULL
);

-- 其他表: api_tokens, files, pastes, storage_configs
```

## 九、关键交互流程

### 9.1 文件上传流程

```
用户选择文件
    ↓
index.html Vue: handleFileSelect()
    ↓
选择存储后端 (storageMode)
    ↓
POST /upload (form-data: file + 参数)
    ↓
functions/upload.js 路由到对应存储后端
    ├── Telegram → utils/telegram.js
    ├── R2 → env.R2_BUCKET.put()
    ├── S3 → utils/s3client.js
    ├── Discord → utils/discord.js
    ├── WebDAV → utils/webdav.js
    ├── GitHub → utils/github.js
    └── HuggingFace → utils/huggingface.js
    ↓
写入 KV: {prefix}{id} (metadata 包含文件名、时间戳、存储类型、文件夹路径)
    ↓
返回上传结果到前端
```

### 9.2 文件删除流程

```
admin.html: 点击删除 → showConfirm → deleteAdminItem('files', fileId)
    ↓
POST /api/manage/delete/[id]
    ↓
functions/api/manage/delete/[id].js
    ├── 从 KV 读取 metadata
    ├── 根据 storageType 调用对应后端删除
    │   ├── R2 → env.R2_BUCKET.delete()
    │   ├── S3 → s3.deleteObject()
    │   ├── Discord → deleteDiscordMessage()
    │   ├── Telegram → Telegram API delete
    │   └── KV → env.img_url.delete()
    └── 清理短链接映射
```

### 9.3 认证与权限校验流程

```
请求到达 → admin/_middleware.js
    ↓
ensureDefaultAdmin() — 首次创建默认管理员
    ↓
isAuthRequired() — 是否配置了 BASIC_USER/PASS
    ├── 是 → checkAuthentication()
    │   ├── 读取 Cookie: k_vault_session
    │   ├── 从 KV 验证 session:{token}
    │   ├── 有效 → next()
    │   └── 无效 → 401
    └── 否 → 检查 KV 是否有用户数据
        ├── 有用户 → 同上认证
        └── 无用户 → 放行（首次设置）
```
