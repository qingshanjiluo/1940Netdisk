### 3.8 index.html — loadSessionUser 方法 (行 2062-2077)

```javascript
loadSessionUser: function() {
  try {
    var sessionId = localStorage.getItem('1940netdisk_session') || sessionStorage.getItem('1940netdisk_session');
    if (!sessionId) { this.sessionUser = null; return; }
    var raw = localStorage.getItem('1940netdisk_data');
    if (!raw) { this.sessionUser = null; return; }
    var data = JSON.parse(raw);
    var users = data.users || [];
    for (var i = 0; i < users.length; i++) {
      if (users[i].id === sessionId) {
        this.sessionUser = users[i];
        return;
      }
    }
    this.sessionUser = null;
  } catch(e) { this.sessionUser = null; }
}
```

**注意**: 此方法使用 localStorage 读取用户数据，与 admin.html 使用 API 调用不同。

## 四、最大嫌疑分析

### 嫌疑 #1: admin.html Vue 未正确挂载（最可能）

**现象解释**: 如果 Vue 未挂载，所有绑定都失效：
- `v-if="currentAdminUser"` → 用户头像不显示（但实际看到的是显示了下拉菜单？）
- `@click="createFolder"` → 无反应
- `@click="selectFolder"` → 无反应
- 左侧面板拉长 → 纯 HTML fallback 显示

**检查方法**: 在浏览器控制台检查 `document.querySelector('#app').__vue__` 是否存在。

### 嫌疑 #2: showPrompt/showToast 函数未定义

如果 admin.html 开头的 `showPrompt`（行 1253）、`showToast` 等全局函数因某处 JS 报错而未定义，则：
- `createFolder` 调用 `showPrompt` 会抛出错误 → 功能无效
- 其他按钮调用 `showToast` 也会失败

**检查方法**: 浏览器控制台输入 `typeof showPrompt` 检查。

### 嫌疑 #3: index.html authChecking 状态异常

`v-if="!authChecking && !isGuest && sessionUser"` 中：
- 如果 `authChecking` 在 mounted 后一直为 true → 整个下拉菜单不显示
- 如果 `authChecking` 短暂变为 false 后又被设为 true → 下拉闪烁

**检查方法**: 在 Vue DevTools 中观察 `authChecking` 值的变化。

### 嫌疑 #4: uploadedBy 错误

```
POST https://29f99dd4.1940netdisk.pages.dev/upload 500 (Internal Server Error)
(index):1869 Uncaught (in promise) ReferenceError: uploadedBy is not defined
```

**位置**: index.html 部署版本的 1869 行
**来源代码**: `functions/upload.js:69` → `const uploadedBy = await getCurrentUploader(context);`
**可能原因**:
1. Cloudflare Pages 部署的 index.html 行号与源文件不同，1869 行可能对应不同代码
2. 后端 `getCurrentUploader` 函数在 KV binding 不可用时抛出异常
3. 编译/压缩过程中引入的代码错误
